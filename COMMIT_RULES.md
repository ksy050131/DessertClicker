# DessertClicker 커밋 규칙

## 커밋 메시지 구조

```
[타입] 제목
```

예: `[feat] 메인 젤리 클릭 기능 구현`

## 타입 정의

| 타입 | 용도 | 예시 |
|---|---|---|
| `[feat]` | 새로운 기능 추가 (스크립트, UI 배치 등) | `[feat] 업그레이드 상점 기능 추가` |
| `[design]` | 이미지, 폰트, 도트 아트 등 디자인 리소스 추가/수정 | `[design] 젤리 도트 스프라이트 추가` |
| `[fix]` | 버그 수정 (에러 해결 등) | `[fix] Input System 에러 해결` |
| `[refactor]` | 코드 리팩토링 (기능 변화 없이 구조만 개선) | `[refactor] 점수 계산 로직 분리` |
| `[docs]` | 문서 수정 (README.md 등) | `[docs] README 작성` |
| `[chore]` | 빌드 설정, 폴더 구조 변경 등 단순 작업 | `[chore] .gitignore 설정` |

## 규칙

- 모든 커밋 메시지는 **한글**로 작성
- 제목은 **명령형**으로 간결하게 (50자 이내)
- 여러 종류의 변경이 섞인 경우, 가장 중요한 타입 사용
